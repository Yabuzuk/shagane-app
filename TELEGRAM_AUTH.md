# Настройка авторизации через Telegram

## Шаги для настройки:

### 1. Создайте Telegram бота

1. Откройте Telegram и найдите [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newbot`
3. Введите имя бота (например: "ШАГАНЭ Delivery")
4. Введите username бота (например: "shagane_delivery_bot")
5. Сохраните полученный токен

### 2. Настройте домен для бота

1. Отправьте команду `/setdomain` в @BotFather
2. Выберите вашего бота
3. Введите домен вашего приложения (например: `https://yourdomain.com`)
   - Для локальной разработки можно использовать `http://localhost:5173`

### 3. Обновите код приложения

В файле `src/pages/LoginPage.jsx` замените:

```javascript
script.setAttribute('data-telegram-login', 'YOUR_BOT_USERNAME');
```

На:

```javascript
script.setAttribute('data-telegram-login', 'shagane_delivery_bot'); // ваш username бота
```

### 4. Запустите приложение

```bash
npm run dev
```

## Как это работает:

1. Пользователь видит экран входа с кнопкой Telegram
2. При клике открывается окно авторизации Telegram
3. После подтверждения данные пользователя сохраняются
4. Пользователь получает доступ к каталогу и может оформлять заказы

## Данные пользователя:

После авторизации доступны:
- `id` - Telegram ID
- `first_name` - Имя
- `last_name` - Фамилия (опционально)
- `username` - Username (опционально)
- `photo_url` - Аватар (опционально)
- `auth_date` - Дата авторизации

## Безопасность:

⚠️ В продакшене обязательно проверяйте подпись данных на сервере!

Telegram отправляет `hash` для верификации данных. Проверка должна выполняться на бэкенде с использованием токена бота.

## Для продакшена:

1. Настройте HTTPS
2. Добавьте серверную валидацию
3. Храните токен бота в переменных окружения
4. Реализуйте JWT токены для сессий
